The Internet needs a format in order to communicate. The TCP/IP solves this problem. It serves as the language of communication of the internet. The Intenet has two layers consisting of a higher layer and a lower layer. "The higher layer, Transmission Control Protocol, manages the assembling of a message or file into smaller packets that are transmitted over the Internet and received by a TCP layer that reassembles the packets into the original message.The lower layer, Internet Protocol, handles the address part of each packet so that it gets to the right destination. Each gateway computer on the network checks this address to see where to forward the message. Even though some packets from the same message are routed differently than others, they'll be reassembled at the destination."
The protocol uses a client/server mode of communication. In which a computer user (a client) requests and is provided a service (such as sending a Web page) by another computer (a server) in the network. TCP/IP communication is primarily point-to-point, meaning each communication is from one point (or host computer) in the network to another point or host computer. (Rouse, 2015)
As discussed in the brief overview, the protocol traces its root from the R&D done by DARPA. After starting the initial ARPANET in 1969, DARPA started working on other data transmission technologies. Eventually in 1982, the US Department of Defense adopted the protocol as a standard for all military communications. Furthermore, corporations with the likes of IBM and AT&T decided to adopt it as well. Since then it has formed the backbone of inter-computer communication.
